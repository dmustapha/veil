# Lessons — Stellar Hacks: Real-World ZK

## 2026-06-25 — Ideas must be ecosystem-native + inventive, not TradFi-with-ZK-bolted-on
Dami rejected V5 and V6 pools (and showed a screenshot of the SAME rejection from the Lepton hackathon). The repeated mistake: every idea imported a traditional-finance problem (solvency, FX honesty, private credit, revenue waterfalls, RWA distribution, bridge security, keeper accountability) and bolted ZK on. "Real-world money welcome" in the brief is a tie-breaker, NOT a mandate to solve a bank's problem.

Fix going forward:
- Generate ONLY from problems native to the crypto/Stellar ecosystem (exist because of how the ecosystem works): sybil/airdrop farming, wash trading / fake volume / mercenary liquidity, missing native primitives (no native VRF), spoofable on-chain activity, transparent-ledger reflexivity, etc.
- Be genuinely INVENTIVE, not a clone of a known pattern.
- MINE the provided sources (repo/chat, Stellar #zk-chat + ecosystem repos + SDF ZK use-cases, Dami's own repos as springboards). Do not invent problem territories from my own head.
- Hard kill at generation: "Would a bank have this exact problem? If yes, wrong frame."
- Still holds from earlier: ZK load-bearing; no front-running (dead on Stellar); no charity/aid/remittance/insurance; no bolted-on AI; no governance (Dami rejected SilentGauge); self-duplication is allowed (intertwining repos welcome); uniqueness = vs other projects.

V5 winner (SilentGauge) and all V6 ideas are CANCELLED.
